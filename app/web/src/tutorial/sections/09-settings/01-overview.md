# Settings — overview

The Settings page covers admin auth, session defaults, theme,
keyboard shortcuts, and the audit log. None of it is per-day
hot-path stuff — most operators visit once at install and never
again.

![Settings page](/tutorial/settings-layout.png)

## What's where

| Sub-section | Purpose |
|---|---|
| **Auth** | Admin password rotation, integration token list link |
| **Session defaults** | Idle threshold + watcher interval applied to every session |
| **Theme** | Dark / light / system |
| **Keyboard shortcuts** | Customise the `g s` / `g c` / etc. map |
| **Audit log** | Search + filter the every-admin-action append-only log |
